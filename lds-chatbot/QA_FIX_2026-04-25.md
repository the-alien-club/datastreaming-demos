## Executive Summary — read this first

The basePath sweep is fixed. Every CRUD UI flow now reaches its API (verified: MCP create succeeds and shows "MCP created" toast). Sign-out now goes to `/agents/sign-in` instead of 404'ing. The PUT `/api/agents/{id}` 500 on stringified `steps` is closed with defensive coercion. The OpenAI-compat path now persists user and assistant messages. Default model in code/UI is now `gpt-4.1-mini` everywhere (a real slug on the platform).

**Two material findings beyond the original sweep:**

1. **NEW BUG (FIXED)**: The platform's `/ai-models` response shape changed. It used to return an expanded `provider: { id, slug, name }` object; it now returns only `providerId: number`. Every page that renders a model picker did `m.provider.name` and crashed the whole page with "can't access property 'name', e.provider is undefined" — the New Agent / Edit Agent / New Specialist / Edit Specialist forms were all unreachable. Fixed by updating the type and using a new `providerLabelFromModel(m)` helper that derives a label from the model's apiUrl host. Verified working in v0.1.9.

2. **P0-2 IS NOT FIXED**: chat is still empty even with a known-good model slug (`gpt-4.1-mini`). The platform workflow runs to completion (HTTP 200, terminal `done` event) but **emits zero content chunks** — `result.stream.agent.chunks.length === 0` for the full duration. This is reproducible from a freshly-created agent with no subagents and no MCPs (the simplest possible deepAgent-only graph). It is **not** a chatbot code bug; it is a backend issue in `api.alien.club` workflow execution (or the `deepAgent` node implementation upstream). v0.1.9 ships with one-line debug logging on the OpenAI-compat path so the empty-content runs can be diagnosed from prod logs without redeploying.

**Bottom line**: This demo is closer to client-ready than it was, but the chat itself is still broken. Sign-in works, every CRUD flow works, agents can be created and edited from the UI, conversations get persisted — but the actual AI response is empty. **Do not show this to a client until the platform team fixes deepAgent execution.**
