# i18n Rule

## Rule

The app uses `next-intl` with `messages/fr.json` and `messages/en.json`. Every
user-visible string in JSX is a translation key. No exceptions.

**French is the default and primary locale.** The product's users are BnF
librarians and scholars; their working language is French. English is a
secondary locale for non-French team members, demos, and documentation.

## Key requirements

**Both locale files must stay in sync.** A key present in one and missing
from the other is a broken build waiting to happen. Every time you add a
key to one file, you add it to both in the same commit.

**Default locale = `fr`** (`as-needed` strategy in `next.config.ts`).
Placeholder strings (`"TODO"`, copied English text) in `fr.json` are not
acceptable.

### Established top-level namespaces

| Namespace | Covers |
|---|---|
| `common` | Shared cross-feature: save, cancel, delete, loading, error, tryAgain |
| `nav` | Step navigation labels (Constituer / Ingérer / Rechercher / Carnet) |
| `projects` | Project list, creation |
| `corpus` | Step 1 — corpus comprehension, chat, intro |
| `ingest` | Step 2 — pipeline stages, status, the "come back later" banner |
| `research` | Step 3 — chat, atelier, carnet, intro |
| `memory` | Memory dialog (both scopes), sections, origins |
| `citations` | Citation side panel and external link labels |
| `tools` | Tool labels for the `BadgeToolCall` chip |

Do not introduce a new top-level namespace unless adding a wholly new feature
domain.

## Key naming

camelCase. Names describe the concept, not the value.

```jsonc
// ✅
"corpus": { "intro": { "gotIt": "J'ai compris" } }

// ❌ snake_case
"corpus": { "intro": { "got_it": "J'ai compris" } }

// ❌ describes the value
"corpus": { "intro": { "iUnderstandThisIntro": "J'ai compris" } }
```

## Scoping `useTranslations`

Always scope to the smallest relevant namespace.

```tsx
// ✅
export function CardCorpusSummary({ corpus }: Props) {
  const t = useTranslations("corpus.summary")
  const tCommon = useTranslations("common")
  return (
    <Card>
      <CardHeader><CardTitle>{tCommon("loading") /* example */}</CardTitle></CardHeader>
      <CardContent>{t("total", { count: corpus.total })}</CardContent>
    </Card>
  )
}

// ❌ root scope pulls the entire tree
const t = useTranslations()

// ❌ page-level scope inside a card component
const t = useTranslations("corpus")
// then: t("summary.total") — defeats scoping
```

## Translating dynamic BnF content

User-visible BnF content from the database (document titles, author names,
note bodies, OCR excerpts, memory items) is **not** translated. It is rendered
as-is. The rule governs static UI chrome — labels, buttons, headings, empty
states, error messages, placeholders, tooltips.

The agent's streamed responses are **not** translated either — they come back
from Claude in French already, per [agent-streaming.md](agent-streaming.md)
and the prompts in [doc 08](../design/docs/08-prompting.md).

## Tool labels for chat chips

The `BadgeToolCall` component reads its label from `tools.<tool_name>`:

```tsx
// components/badges/tools/call.tsx
const t = useTranslations("tools")
return <Badge>{t(toolName)} · via MCP</Badge>
```

Every tool name in `lib/agent/tools.ts` (see [constants.md](constants.md)) has
a matching key in `tools.*`. New tools added to the dispatcher require both
files updated together.

## Onboarding intros

The corpus and research intro dialog copy lives in `corpus.intro.*` and
`research.intro.*`. The text supports markdown (rendered with a safe
markdown component) so the design team can iterate without code changes.
Keep keys stable across copy revisions — only the values change.

## Adding a feature

1. Identify the feature namespace (`corpus`, `ingest`, `research`, …).
2. Identify the component sub-namespace (`summary`, `filters`, `intro`, `dialog`).
3. Add all keys to `messages/fr.json` under `feature.component.*` with proper
   French.
4. Add the same keys to `messages/en.json` (translation can lag in quality but
   never in coverage).
5. Use `useTranslations("feature.component")` — never the root
   `useTranslations("feature")`.

## Anti-patterns (FORBIDDEN)

```tsx
// ❌ Hardcoded French in JSX
<Button>Constituer un corpus</Button>

// ❌ Hardcoded placeholder
<Input placeholder="Posez votre question…" />

// ❌ Root-level useTranslations
const t = useTranslations()

// ❌ Flat top-level key
// messages/fr.json: { "createProjectButton": "Créer" }

// ❌ Key in fr.json missing in en.json (or vice versa)
```

## Scope

Applies to all files under `app/` and `components/`. Dynamic strings from the
database are rendered as-is — they are not translation keys.
